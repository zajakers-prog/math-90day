# 90일 수학 완성 코스 - 개선 프롬프트

## 현재 상태 (이미 완성된 것)

React + Vite + Tailwind 기반 초등 수학 학습 앱이 있다.

### 핵심 구조:
- **2학년 과정** (1학년 복습): 덧셈/뺄셈 10~20 이내, 18주 커리큘럼
- **4학년 과정** (3학년 복습): 사칙연산, 3자리수까지, 18주 커리큘럼
- 매일 30문제 (계산 27 + 추리 3)
- 날짜 기반 시드로 매일 같은 문제 보장 (2026.3.9 시작)
- 영업일(월~금) 기준 Day 카운트
- 정답/오답 피드백, 점수, 진행률 표시
- 완료 화면에 퍼센트 + 격려 메시지

### 기술 스택:
- React 18, Vite 5, Tailwind 3, Lucide React
- 단일 컴포넌트 (`src/App.jsx`) 구조
- seededRandom 함수로 일관된 문제 생성

## 개선해야 할 것

### 1. 2학년 과정 문제 범위 수정
현재 2학년 `range`가 너무 작거나 커리큘럼 진행이 비현실적임.
- week 1~2: 덧셈인데 `rand1 % 10`으로 고정 → range를 제대로 활용해야 함
- week 3~4: 뺄셈인데 `rand1 % 15`로 처리 → 1학년 수준에 맞게 조정
- mixed 주차들: 덧셈/뺄셈 섞어야 하는데 분기 로직이 부족
- 복습/최종복습 주차: 전부 `range: [1, 20]`으로 동일 → 변화 없음

**수정 방향**: 1학년 교과서 기준으로 범위 재설계. 10 이내 → 받아올림 없는 20 이내 → 받아올림 있는 덧셈 순서.

### 2. 4학년 과정 Phase 3~6 상세화
week 8~18이 전부 mixed/review/final로 뭉뚱그려져 있음.
- mixed 주차: 어떤 연산 조합인지 구분 없음
- review 주차: 이전 주차 내용을 실제로 반영하지 않음
- final 주차: 6주나 되는데 전부 같은 로직

**수정 방향**: 각 주차별로 출제 범위와 연산 비율을 다르게 설정. 예: final-1은 덧뺄셈 위주, final-3은 나눗셈 위주 등.

### 3. 추리 문제(REASONING_BANK) 학년별 분리
현재 추리 문제가 학년 구분 없이 10개 공용.
- 2학년한테 `□ ÷ 3 = 6`이나 `1², 2², 3²` 패턴은 너무 어려움
- 4학년한테 `5 + ? = 12`는 너무 쉬움

**수정 방향**:
```js
const REASONING_BANK_2 = [...]; // 10개+, 1학년 수준 (10 이내 수, 단순 패턴)
const REASONING_BANK_4 = [...]; // 10개+, 3학년 수준 (곱셈/나눗셈 포함)
```

### 4. 문제 출력 형식 통일
현재: `23 + 45 = ?`
이걸 세로 계산식으로도 보여주면 학습 효과 증가 (특히 받아올림/받아내림).

### 5. "계속" 기능
학습 중간에 나갔다 돌아왔을 때 이어서 할 수 있도록:
- localStorage에 진행 상태 저장
- 오늘 날짜 + 모드별로 `currentIndex`, `score` 저장/복원

### 6. "다시" 기능
오늘 문제를 처음부터 다시 풀 수 있는 버튼. 점수 리셋.

### 7. 오답 노트
틀린 문제를 모아서 세션 끝에 복습할 수 있게.
- 완료 화면에서 틀린 문제 목록 표시
- "오답 다시 풀기" 버튼

### 8. UI/UX 개선
- 모바일 터치 최적화 (숫자 키패드 자동 표시)
- 문제 전환 애니메이션
- 다크모드 (선택)

## 핵심 코드 참고

시드 기반 랜덤:
```js
function seededRandom(seed) {
  const x = Math.sin(seed) * 10000;
  return x - Math.floor(x);
}
```

날짜 계산 (영업일 기준):
```js
const startDate = new Date(2026, 2, 9); // 2026년 3월 9일 시작
// 월~금만 카운트, 최대 90일
```
